Черновик для цикла «как устроен IT».

Открыть сайт кажется простейшим действием: написал адрес, нажал Enter, увидел страницу. Но внутри за эту секунду происходит целая маленькая логистика.

Эта статья может собрать вместе предыдущие темы: файлы, программы, интернет, клиент, сервер, данные и инструкции.

# Аналогия

Представим, что ты хочешь попасть в кафе.

Сначала нужно понять, где оно находится. Потом дойти или доехать. Потом попросить меню. Потом кухня что-то приготовит. Потом тебе принесут результат.

Открытие сайта похоже: браузер выясняет адрес сервера, идет к нему по сети, просит страницу, получает ответ и собирает из него то, что ты видишь.

# Браузер

Тезисы для раскрытия:

- браузер - это программа;
- он умеет отправлять запросы и показывать страницы;
- адресная строка - не просто поле ввода, а начало цепочки;
- браузер понимает HTML, CSS и JavaScript;
- он превращает файлы и данные в видимую страницу.

Можно подчеркнуть: сайт не "живет" в браузере, браузер его получает и собирает.

# DNS

Тезисы:

- человеку удобно помнить имя сайта;
- компьютерам нужны адреса;
- DNS помогает найти адрес по имени;
- это похоже на телефонную книгу;
- если DNS не работает, имя есть, но найти нужный сервер не получается.

Не углубляться в записи DNS, только дать модель.

# Запрос и ответ

Тезисы:

- браузер отправляет серверу запрос;
- сервер отвечает данными;
- часто первый ответ - HTML;
- потом браузер может запросить CSS, JS, картинки, шрифты;
- страница собирается из нескольких частей.

Пример:

```text
браузер: дай главную страницу
сервер: вот HTML
браузер: дай стили, скрипты и картинки
сервер: вот они
```

# HTML, CSS, JavaScript

Тезисы:

- HTML - структура: что на странице есть;
- CSS - внешний вид: как это выглядит;
- JavaScript - поведение: что меняется и реагирует;
- браузер соединяет это в интерфейс;
- современные сайты часто догружают данные уже после первого показа.

Аналогия: HTML - каркас дома, CSS - ремонт и мебель, JS - электричество и механизмы.

# Почему сайт может не открыться

Тезисы:

- имя не нашлось;
- сервер недоступен;
- интернет между ними сломался;
- сервер ответил ошибкой;
- браузер получил данные, но скрипт упал;
- проблема может быть на разных участках цепочки.

# Вывод

Когда ты открываешь сайт, браузер не просто "показывает интернет". Он ищет нужный сервер, просит у него данные, получает набор файлов и инструкций, а потом собирает из них страницу.

# Куда дальше

Многие сайты не просто отдают готовые файлы, а хранят пользователей, заказы, сообщения и настройки. Поэтому дальше - зачем нужны базы данных.
