Черновик для цикла «как устроен IT».

Файлы кажутся чем-то очевидным: документ, картинка, музыка, видео, архив. Но если копнуть чуть глубже, файл - это одна из самых важных договоренностей в IT.

Файл позволяет взять кусок данных, сохранить его, перенести, отправить, открыть другой программой и через год снова понять, что там было.

# Аналогия

Представим коробку.

В коробку можно положить что угодно: книгу, тарелку, провода, носки. Но если коробка без подписи, человеку придется открыть ее и разбираться.

Файл похож на такую коробку. Внутри байты. Чтобы понять, что они значат, нужна договоренность: это текст, это картинка, это звук, это таблица.

# Файл как набор байтов

Тезисы для раскрытия:

- внутри файла нет "картинки" или "песни" в человеческом смысле;
- там лежат байты;
- программа читает байты по правилам формата;
- если читать неправильной программой, получится мусор или ошибка;
- расширение файла помогает угадать формат, но не является самой сутью файла.

Пример: `.jpg`, `.txt`, `.mp3`, `.zip`. Расширение как наклейка на коробке.

# Формат

Тезисы:

- формат говорит, как устроены байты внутри;
- текстовый файл читается по одним правилам;
- картинка - по другим;
- архив - по третьим;
- если программа знает формат, она может открыть файл;
- если не знает, ей нечего с ним делать.

Можно добавить мысль: формат - это общий язык между программами.

# Почему файлы портятся

Тезисы:

- если часть байтов потерялась или изменилась, формат может сломаться;
- иногда файл открывается частично;
- иногда не открывается вообще;
- поэтому важны копии и сохранение;
- "битый файл" - это не мистика, а нарушенная структура данных.

Пример: книга с вырванными страницами. Иногда смысл еще можно восстановить, иногда нет.

# Файлы и папки

Тезисы:

- папки нужны не компьютеру как идея, а людям и программам для порядка;
- путь к файлу помогает его найти;
- операционная система следит, где что лежит;
- разные программы могут читать и писать файлы, если у них есть права.

Не уходить в файловые системы глубоко, только подготовить тему ОС.

# Вывод

Файл - это способ сохранить данные в переносимой форме. Но сам по себе файл ничего не объясняет. Смысл появляется, когда программа знает формат и умеет прочитать байты правильно.
