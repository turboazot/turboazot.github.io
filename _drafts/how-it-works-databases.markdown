Черновик для цикла «как устроен IT».

Можно хранить данные в файлах. Тогда зачем нужны базы данных?

Пока данных мало и с ними работает один человек, файла часто хватает. Но когда данные нужно быстро искать, менять, связывать, защищать и обслуживать одновременно для многих пользователей, обычная куча файлов быстро превращается в боль.

# Аналогия

Представим библиотеку.

Если у тебя дома 20 книг, можно просто поставить их на полку. Если книг 20 миллионов, нужны каталоги, индексы, правила выдачи, учет, сотрудники и система, которая понимает, где что лежит.

База данных - это не просто место, куда сложили данные. Это организованная система работы с ними.

# Почему не просто файлы

Тезисы для раскрытия:

- файл хорошо хранит кусок данных;
- но в файлах неудобно быстро искать по сложным условиям;
- неудобно безопасно менять одну часть, пока другие тоже работают;
- трудно следить за связями;
- трудно гарантировать порядок, если много пользователей пишут одновременно.

Пример: интернет-магазин с товарами, пользователями, корзинами, заказами и оплатами.

# Таблицы и связи

Тезисы:

- данные часто раскладывают по сущностям: пользователи, товары, заказы;
- у сущностей есть поля;
- между ними есть связи;
- заказ связан с пользователем и товарами;
- база помогает не хранить все одной кашей.

Не обязательно углубляться в SQL, но можно показать идею таблицы как аккуратной ведомости.

# Поиск

Тезисы:

- база умеет быстро находить нужное;
- для этого используются индексы;
- индекс похож на оглавление или алфавитный указатель;
- без индекса пришлось бы просматривать все подряд;
- поэтому база может быстро отвечать на вопросы типа "найди все заказы этого пользователя".

# Одновременная работа

Тезисы:

- в реальном сервисе много людей делают действия одновременно;
- двое могут купить последний товар;
- пользователь может платить, а сервер в этот момент обновляет заказ;
- база помогает сохранять согласованность;
- иначе данные могут разъехаться.

Можно объяснить через кассу и склад: если не синхронизироваться, один и тот же товар продадут дважды.

# Вывод

База данных нужна не потому, что файлы плохие. Она нужна, когда данные становятся живой частью системы: их много, они связаны, их постоянно ищут, меняют и защищают.

# Куда дальше

Дальше стоит разобрать API: как программы просят друг друга прочитать, изменить или отдать данные по понятным правилам.
