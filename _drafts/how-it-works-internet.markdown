Черновик для цикла «как устроен IT».

Интернет часто воспринимается как место: "залить в интернет", "найти в интернете", "интернет сломался". Но технически интернет больше похож на систему доставки сообщений между компьютерами.

Не облако, не эфир и не магический туман. Просто много устройств, которые умеют пересылать данные друг другу по правилам.

# Аналогия

Представим почту.

Ты пишешь письмо, указываешь адрес, отдаешь его в доставку. Письмо может пройти через несколько отделений, сортировок и машин. Тебе не обязательно знать весь маршрут. Главное, чтобы у системы были адреса и правила доставки.

Интернет похож на почту, только письма разбиваются на маленькие куски, летят очень быстро и собираются обратно.

# Компьютеры в сети

Тезисы для раскрытия:

- в интернете общаются устройства: телефоны, ноутбуки, серверы, роутеры;
- у устройств есть адреса;
- данные передаются не "целиком магией", а кусками;
- разные устройства помогают этим кускам добраться до цели;
- интернет - это сеть сетей.

Важно не перегружать деталями IP/TCP, но можно назвать их как правила доставки.

# Пакеты

Тезисы:

- большой кусок данных делится на маленькие пакеты;
- у пакета есть служебная информация: откуда, куда, что это за часть;
- пакеты могут идти разными маршрутами;
- на другой стороне их собирают обратно;
- если что-то потерялось, некоторые протоколы умеют попросить отправить заново.

Аналогия: разобрать шкаф на детали, отправить коробками, собрать на месте.

# Сервер и клиент

Тезисы:

- клиент обычно просит;
- сервер обычно отвечает;
- браузер просит страницу;
- приложение просит данные;
- сервер может обслуживать много клиентов.

Не делать из этого жесткую догму: один компьютер может быть клиентом в одном разговоре и сервером в другом.

# Почему интернет иногда тормозит

Тезисы:

- пакеты идут через много промежуточных точек;
- где-то может быть перегруз;
- сервер может отвечать медленно;
- Wi-Fi может быть слабым;
- расстояние и маршруты тоже влияют;
- "интернет тормозит" может означать несколько разных проблем.

# Вывод

Интернет - это не место, а договоренность о том, как компьютерам находить друг друга и обмениваться сообщениями. Почти все современные сервисы строятся поверх этой простой идеи: кто-то отправляет запрос, кто-то отвечает.

# Куда дальше

Дальше можно взять самый знакомый пример интернета: что происходит, когда ты вводишь адрес сайта в браузере и нажимаешь Enter.
